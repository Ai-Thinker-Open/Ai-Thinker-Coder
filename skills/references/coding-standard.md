# 嵌入式产品研发C语言编码规范要求

> 来源：安信可科技内部编码规范（下属执行版）  
> 完整规范文档：[coding-standard.md](./coding-standard.md)  
> 适用：所有安信可嵌入式产品 C 语言代码编写、修改、评审及维护

---

## 一、核心要求：axk 函数头规范（必严格执行）

所有 C 语言函数（全局、静态、中断、工具函数无例外），必须在函数定义/声明开头添加以"axk"为前缀的函数头注释。

### 1.1 函数头格式（固定模板）

```c
/*
 * axk_函数功能简述（简洁，1句话，不超过50字）
 * @brief  函数详细描述（补充功能细节、设计思路，可选，复杂函数必写）
 * @param  参数名1：参数说明（明确类型、含义、取值范围，无参数则写"无"）
 * @param  参数名2：参数说明（多个参数依次排列，参数名与函数定义一致）
 * @return 返回值说明（明确类型、含义、不同返回值对应场景，无返回值则写"无"）
 * @note   注意事项（可选，如使用限制、调用前提、异常处理说明、关联函数等）
 * @author 开发者姓名
 * @date   开发/修改日期（格式：YYYY-MM-DD）
 */
返回值类型 axk_函数名(参数列表)
{
    // 函数体
}
```

### 1.2 关键规则

| 类型 | 规则 |
|------|------|
| **函数名** | 必须以 `axk_` 开头，小写字母+下划线，见名知意 |
| **全局变量** | 以 `g_axk_` 为前缀 |
| **静态变量** | 以 `s_axk_` 为前缀 |
| **局部变量** | 无前缀，避免单字母（i/j/k 循环变量除外） |
| **宏定义** | 全大写+下划线，以 `AXK_` 为前缀 |
| **枚举/结构体** | 以 `Axk` 为前缀（CamelCase），成员全大写+下划线 |

### 1.3 静态函数与中断函数特殊要求

- **静态函数**：`@note` 中注明"静态函数，仅本文件内可调用"
- **中断服务函数**：`@brief` 注明中断触发方式，`@note` 注明中断优先级及处理注意事项

### 1.4 正确示例

```c
/*
 * axk_uart_init：初始化串口外设
 * @brief  配置串口波特率、数据位、停止位、校验位，使能接收/发送功能
 * @param  baud：波特率，取值 9600/19200/38400/115200
 * @return 无
 * @note   1. 调用前需确保串口外设已使能；2. 波特率需与外设匹配
 * @author 张三
 * @date   2026-04-27
 */
void axk_uart_init(int baud)
{
    // 函数体
}
```

## 二、代码格式规范

### 2.1 缩进与空格

- **缩进**：4 个空格，禁止 Tab 键
- **大括号**：`{` 紧跟语句末尾，`}` 单独占一行对齐
- **运算符**：前后各加 1 空格，如 `a + b`、`c == d`
- **逗号**：后加 1 空格，如 `func(a, b)`
- **括号**：`()` 前后不加空格，如 `if (a == b)`

### 2.2 空行与换行

- 函数之间空 **2 行**
- 函数内部不同逻辑块之间空 **1 行**
- 每个语句**单独占一行**
- if/for/while 即使单行函数体也**必须加 `{}`**

### 2.3 注释规范

- 关键逻辑、复杂算法、异常处理处添加行注释 `// 注释内容`
- **禁止**无意义注释（如 `// 循环`）
- **禁止**注释掉的无用代码（无用代码直接删除）
- `.h` 文件开头需文件说明注释（用途、函数/宏、作者、日期）

## 三、安全性规范

### 3.1 参数校验

所有函数输入参数必须校验合法性（是否为空、取值范围），防止非法参数导致崩溃。

### 3.2 内存管理

- 嵌入式环境**避免动态内存分配**（malloc/free）
- 必须使用时严格控制大小，分配成功后及时释放
- **禁止**野指针、空指针

### 3.3 中断安全

- 中断服务函数**禁止调用耗时函数**
- 中断中**禁止大量数据处理**，尽量缩短执行时间
- 中断与主程序共享变量需加 `volatile` 修饰

### 3.4 宏定义安全

多表达式宏必须用括号包裹：
```c
#define AXK_ADD(a, b) ((a) + (b))
```

## 四、可维护性规范

| 要求 | 说明 |
|------|------|
| 函数拆分 | 单函数功能单一，代码不超过 100 行（特殊场景除外） |
| 代码复用 | 相同功能封装为工具函数，避免重复编码 |
| 兼容性 | 避免直接使用硬件寄存器地址，优先使用封装宏，便于移植 |

## 五、文件组织规范

### 5.1 文件命名

- `.c`、`.h` 文件以 `axk_` 为前缀，小写+下划线，如 `axk_uart.c`

### 5.2 头文件防重复包含

```c
#ifndef AXK_UART_H
#define AXK_UART_H
// ...
#endif
```

### 5.3 文件内容顺序

**.c 文件**：`头文件包含 → 宏定义 → 全局变量 → 静态函数声明 → 函数定义`  
**.h 文件**：`防重复包含宏 → 宏定义 → 枚举/结构体 → 函数声明`

- [ ] 函数修改后 @note 中注明修改人、日期及修改内容

### 6.2 通用编码规范
- [ ] 命名符合前缀及格式要求
- [ ] 4 空格缩进，大括号/空行/空格/换行符合规范
- [ ] 关键逻辑有行注释，无无意义注释，无注释无用代码
- [ ] 参数校验、内存安全、中断安全、宏定义安全符合要求

### 6.3 文件组织规范
- [ ] .c/.h 文件命名以 `axk_` 为前缀
- [ ] .h 文件有防重复包含宏
- [ ] 文件内容顺序符合规范
